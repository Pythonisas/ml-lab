** Objetivo
Construir un mini‑sistema *reproducible* que estime el *precio* de una vivienda a partir de características simples y que se pueda ejecutar desde la terminal.

#+begin_quote
Esto NO va de “adivinar el precio perfecto”.
Va de aprender el flujo real: datos → preprocesado → modelo → evaluación → guardar → predecir.
#+end_quote

** Forma de trabajo
- Trabajo en *tríos*.
- Ejecutable: *terminal* (scripts Python).
- Se valora: robustez, claridad, reproducibilidad.

** Entregables
- Carpeta =ml-vivienda/= con la estructura indicada.
- Scripts: =src/train.py=, =src/predict.py=, =src/features.py=, =src/utils.py=.
- Dataset: =data/viviendas.csv= (ideal: 200–500 filas) y =README.org=.
- Al entrenar: =models/model.joblib= y =models/metadata.json=.
- =requirements.txt= con versiones fijas.

** Flujo recomendado
1. *Carga y validación*: columnas obligatorias, tipos y nulos.
2. *EDA breve*: distribuciones, outliers, correlaciones; histograma de =precio=.
3. *Preprocesado*:
   - Numéricas: =SimpleImputer(strategy='median')= + (opcional) =StandardScaler=.
   - Categóricas: =OneHotEncoder(handle_unknown='ignore')=.
   - Pipeline: =ColumnTransformer= + =Pipeline=.
   - Considerar =log(precio)= si está sesgado.
4. *Modelado*:
   - =LinearRegression= (baseline) y =RandomForestRegressor=.
   - Añadir =Ridge= como alternativa.
   - Hiperparámetros con =RandomizedSearchCV= (opcional).
5. *Evaluación*:
   - =train_test_split= 80/20 con =random_state=.
   - Reportar MAE, RMSE y R².
   - CV 5-fold; reportar media ± std.
6. *Persistencia y metadata*:
   - Guardar mejor modelo con =joblib=.
   - =metadata.json= con fecha, versión sklearn, features, métricas.
7. *Predict CLI*:
   - =argparse= + fallback =input()=.
   - Validar entradas; manejar categorías desconocidas.

** Rúbrica (10 + bonus)

* Guía extra — Conseguir un dataset real (Comunidad de Madrid)
** Resumen
La Comunidad de Madrid publica conjuntos de datos en su *Catálogo de Datos Abiertos (CKAN)* y muchos aparecen también en =datos.gob.es=.

** Fuentes principales
| Fuente | Cobertura | Formatos | Acceso |
|---|---:|---|---|
| Catálogo Comunidad de Madrid (CKAN) | Toda la Comunidad | CSV; JSON; ZIP | Descarga directa; API CKAN |
| datos.gob.es | Agregador nacional | CSV; JSON | Fichas con enlaces directos y licencia |

** Pasos prácticos
1. Buscar en el Catálogo (=datos.comunidad.madrid=) o en =datos.gob.es= con: =viviendas=, =transacciones=, =AVS=.
2. Revisar ficha técnica: columnas, frecuencia, licencia.
3. Descargar CSV/JSON (enlace de download o API CKAN).
4. Preprocesar: normalizar columnas, filtrar por municipio/distrito, guardar copia con fecha.

** Ejemplo de descarga rápida (plantilla)
#+begin_src bash
curl -L -o viviendas_cam.csv "https://datos.comunidad.madrid/dataset/.../download/viviendas.csv"
#+end_src
